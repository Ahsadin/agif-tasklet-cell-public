# 07 Test Strategy

## Levels

- Unit: parsing/validation helpers.
- Integration: load/verify/run flow.
- Conformance: verifier pack pass/fail behavior.
- Fuzz/property tests: schema boundaries and unsafe input patterns.

## Determinism expectations

- Prefer canonical JSON output comparisons.
- Use explicit tolerance only where needed.
- Keep deterministic error codes stable across runs.

## Setup-phase checks

- Docs completeness checks.
- Workflow rule grep checks.
- No product code guard in scaffold directories.

## Phase 2 Replay Checklist

- Validate each fixture before `run` when required by conformance gating.
- Re-run at least one rejection case twice to confirm deterministic error code/message stability.
- For enforcement report checks, confirm `offline_level`, `filesystem_level`, `limits_applied`, and `limits_enforcement` fields are present.
- Record command, JSON output, and exit code in the issue snapshot.

## Phase 4 Personal Layer gate checklist (MVP-5)

- Validate feedback/proposal payload against `personalization.schema.json` before apply.
- If schema validation fails, return a fail-closed deterministic rejection.
- Run all cases in `local_regression/` for the same `cell.id + cell.version + bundle_hash` identity key.
- If any local regression case fails, return a fail-closed deterministic rejection and do not apply personalization.
- Apply is allowed only when core verifier checks and local regression checks both pass.
- Enforce max personalization storage before apply and reject deterministically when exceeded.
- Enforce max learning compute per day before apply and reject deterministically when exceeded.
- Enforce max proposals per day before apply and reject deterministically when exceeded.
- Retain only the configured number of rollback snapshots and keep snapshot index deterministic.

## Finance Desk conformance notes (MVP-4)

- Conformance goldens must explicitly cover:
  - finance extraction,
  - completeness checks,
  - VAT math checks,
  - duplicate detection.
- Determinism checks must include strict labels and bounded tolerance only where explicitly declared.
- Replay determinism check must reproduce the same finance case `result_hash` from replay records.
- Gateway no-bypass checks must enforce finance allowlist policy before any runner execution path.

## Finance Desk Personal Layer conformance notes (MVP-5)

- Finance Desk learning fixture must include at least one correction-driven rule in `examples/finance/learning_feedback.json`.
- Apply the learning fixture with `cell learn` before running local regression checks.
- Local regression fixture `examples/finance/local_regression_after_learning_input.json` represents a post-learning replay case for extractor safety checks.
- Core behavior remains valid only if `./runner/cell test cells/finance_doc_extractor_neural_v1.cell` still passes after learning apply.
- Personal Layer learning evidence must remain deterministic across repeated runs for the same fixture payload.
